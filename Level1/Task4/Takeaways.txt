Key Takeaways

Requests = Minimum guarantee (what you need)
Limits = Maximum allowed (what you can't exceed)
Memory limit exceeded = Pod gets killed
CPU limit exceeded = Pod gets throttled (slowed down)
Always set limits in production = Prevents one app from ruining everything

Bottom line: Resource limits are like setting house rules so everyone can live together peacefully! ğŸ 



## Why Use Resource Limits?

### âœ… Benefits:

1. **Fair sharing** - Everyone gets resources
2. **Prevent crashes** - One bad app can't kill others
3. **Predictable costs** - Know how much resources you need
4. **Better scheduling** - Kubernetes knows where to place pods
5. **Stability** - Cluster stays healthy
